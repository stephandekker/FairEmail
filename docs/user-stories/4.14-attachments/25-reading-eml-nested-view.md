## Parent Feature

#4.14 Attachments

## What to build

When a received message contains an EML attachment, allow the user to open it as a fully rendered nested message within the application — displaying the forwarded message's headers, body, and attachments without requiring an external tool.

Covers epic sections: US-38, FR-43, AC-18.

## Acceptance criteria

- [ ] An EML attachment can be opened within the application (AC-18).
- [ ] The opened EML is rendered as a fully formatted nested message with headers, body, and its own attachments.
- [ ] No external tool is required to view the EML content.
