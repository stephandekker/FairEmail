## Parent Feature

#4.14 Attachments

## What to build

Once an attachment is downloaded, enable three actions: (1) open it using the system's default application for that file type, (2) save it to a user-chosen location on disk, (3) share it directly to another application via the system share mechanism — without requiring the user to manually save and locate the file first.

Covers epic sections: US-35, US-36, US-37, FR-40, FR-41, FR-42.

## Acceptance criteria

- [ ] Downloaded attachments can be opened via the system's default application handler (FR-40).
- [ ] Downloaded attachments can be saved to a user-chosen disk location (FR-41).
- [ ] Downloaded attachments can be shared to other applications via the system share mechanism (FR-42).
- [ ] All three actions are available from the attachment list item's context actions.
