# Local vs. Server Results Indicator

## Parent Feature
#5.1 Server-Side IMAP Search

## User Story
As any user, I want a clear indicator in the search results view that tells me whether I am seeing local results or server results, so that I understand the scope of what is displayed.

## Blocked by
2-server-side-search-single-folder

## Acceptance Criteria
- The search results view displays a visible indicator of whether results are from a local search or a server search.
- The indicator is always present when search results are displayed.
- The indicator is distinct and unambiguous (e.g. a label, badge, or header text).

- US-20
- FR-25

## Notes
- This is a small but important UX slice that supports the epic's goal G2 (clear, discoverable path from local to server search) by making the current search scope transparent to the user.
